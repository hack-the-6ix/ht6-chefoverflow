Chef Overflow — Agent API Reference

Build an AI agent to run your kitchen. Top scores earn automatic admission to Hack the 6ix 2026.

← Back to Game KitchenAPI v2.0.0

1. How the Game Works

Five chefs move on a grid kitchen. You send them to stations with KitchenAPI.command(chefId, stationId). Orders appear at reception stands; each order lists ingredients and the exact preparation state required (raw, chopped, or cooked).

Chefs take ingredients from bins, chop at cutting boards, cook on stoves, combine items on plating areas with a clean plate from the dish rack, then deliver the finished plate to the reception stand that holds that order.

After a correct delivery, the customer eats and leaves a dirty dish on the stand. You must bus it (empty hands), wash at the sink, and return clean plates to the dish rack. Plates are limited—running out stalls the kitchen.

The game ends after three expired orders. Your score is the sum of delivery points (plus endurance-time bonus after 10 minutes). Difficulty ramps aggressively: the first minute is approachable manually; after that, automation is essentially required.

Phases (wall-clock): Tutorial (~0–1:00) serves Salad & Steak; Ramp (~1:00–2:30) adds Burger. Automation (2:30–10:00) introduces harder recipes in stages (Pizza → Deluxe Burger → Feast → Supreme) while spawn rate and order timers tighten on smooth curves. Endurance (10:00+) unlocks the full recipe mix with the tightest pacing and bonus score tick.

2. Quick Start

  1. Open index.html in Chrome (or another modern browser).
  2. Open DevTools (F12) → Console.
  3. Paste the Example 1 script from Section 8 below (or the minimal snippet from the console banner after loading the game).
  4. Click Start Game or run KitchenAPI.start().
  5. Watch Chef 0 complete steak orders when tutorial-phase orders include Steak.
  6. Iterate: add chefs, more recipes, bussing, and rush handling.

You can also save agent code in a .js file and inject it via a bookmarklet, a local script tag in a fork of index.html, or a browser extension—any way that runs in the game’s page context so window.KitchenAPI is available.

3. API Reference

KitchenAPI.version

Type: string. Current value: "2.0.0".

KitchenAPI.start()

Starts or restarts the game. Resets stations, chefs, orders, score, and clears all event listeners registered through on / onTick / etc. Re-register hooks after each start.

Returns: undefined.

KitchenAPI.start();

KitchenAPI.togglePause()

Toggles pause. Updates the in-game Pause button label.

Returns: undefined.

KitchenAPI.command(chefId, stationId)

Signature: (chefId: number, stationId: string) => { success: boolean, error?: string }

Queues pathfinding to a walkable tile adjacent to the target station, then runs the same interaction as a mouse click. Fails if the game is not running, is paused, the chef id is invalid, the chef is busy, the station id is unknown, there is no adjacent walkable tile, or A* finds no path.

Chef IDs are 0–4. Station IDs are strings such as 'bin_3', 'stove_0', 'reception_2' (see Section 4).

const r = KitchenAPI.command(0, 'sink_0');
if (!r.success) console.error(r.error);

KitchenAPI.boost(chefId)

Returns: { success: boolean, error?: string }. Applies a 3.5s movement speed boost with a 12s cooldown. Same rules as the B key (but targets a chef by id).

KitchenAPI.selectChef(chefId)

Visual selection only (highlight / UI). Does not move the chef.

Returns: undefined.

KitchenAPI.getState()

Returns a plain snapshot of the game. Ingredient objects are { ingredient, state }. Plates are { type: 'plate', dirty: boolean, items: [...] }. The run ends when failedOrders >= maxFailedOrders (both are in the snapshot; maxFailedOrders is 3).

{ time: number, // seconds elapsed score: number, difficulty: number, // current multiplier (phase-based curve) phase: string, // "tutorial" | "ramp" | "automation" | "endurance" (endurance at time >= 600s) streak: number, bestStreak: number, rush: { active: boolean, timeLeft: number, cooldown: number }, failedOrders: number, maxFailedOrders: number, // game ends when failedOrders >= this (currently 3) running: boolean, paused: boolean, gameOver: boolean, chefs: [{ id: number, // 0–4 name: string, pos: [x, y], holding: null | { ingredient, state } | { type: "plate", items, dirty }, busy: boolean, hasPath: boolean, boostActive: boolean, boostTime: number, boostCooldown: number }], stations: { ingredientBins: [{ id, name, pos, ingredient }], stoves: [{ id, name, pos, cooking, cookTime, maxCookTime, ready, burnt }], cuttingBoards: [{ id, name, pos, busy, processing, processTime, maxProcessTime }], platingAreas: [{ id, name, pos, items }], receptionStands: [{ id, name, pos, order, hasDirtyDish }], dishRacks: [{ id, pos, cleanPlates, dirtyPlates, maxClean }], sinks: [{ id, pos }], trashCans: [{ id, pos }], counters: [{ id, pos, items }] }, orders: [{ id, dish, timeLeft, standId, components }], recipes: object // full internal RECIPES table }

Reception order when present: { id, dish, timeLeft, components }. Stove ready / burnt mirror the UI progress thresholds.

KitchenAPI.getRecipes()

Returns: Array<{ name, emoji, difficulty, components }> — static recipe metadata without internal step strings.

Event registration

Each method below returns an unsubscribe function: const off = KitchenAPI.onTick(fn); off();

  • KitchenAPI.on(eventName, fn) — generic listener.
  • KitchenAPI.onTick(fn) — alias for on('tick', fn).
  • KitchenAPI.onOrderSpawned(fn)
  • KitchenAPI.onOrderExpired(fn)
  • KitchenAPI.onOrderDelivered(fn)
  • KitchenAPI.onOrderFailed(fn)
  • KitchenAPI.onGameOver(fn)
  • KitchenAPI.onPhaseChanged(fn)

Event names for on(): 'tick', 'orderSpawned', 'orderExpired', 'orderDelivered', 'orderFailed', 'gameOver', 'phaseChanged'.

Payloads:

  • tick — { dt: number, time: number }
  • orderSpawned — { id, dish, timeLeft, standId, components }
  • orderExpired — { id, dish, standId }
  • orderDelivered — { id, dish, score, streak } (streak after this delivery)
  • orderFailed — { dish } (wrong plate at reception)
  • gameOver — { score, time, bestStreak }
  • phaseChanged — { phase }

6. Game Mechanics for Agents

  • Chefs block each other — no two chefs on one tile; pathing may fail if the only adjacent tile is occupied.
  • One hold slot — either one ingredient (with state) or one plate (bag of ingredients).
  • command() is asynchronous — it sets path and targetStation. Wait until !chef.busy && !chef.hasPath before issuing a new command for that chef.
  • Busy during processing — chopping, stove wait, and sink wash set busy. Do not stack commands on busy chefs.
  • Stoves keep cooking — if the chef leaves after placing food, the item stays and can burn; send someone back to pick up.
  • Plates are a resource — track stations.dishRacks[0].cleanPlates; bus dirty stands → sink → rack.
  • Delivery matching is exact — component count and each (ingredient, state) must match; extras fail.
  • Wrong delivery — plate is cleared, streak resets to 0.
  • VIP orders — shorter timer, 1.5× score (star in UI).
  • Streak multiplier — +5% per consecutive success, capped at +100% (2×) at streak 20+; wrong delivery or expiry resets streak.

7. Agent Architecture Tips

  • Prove one recipe with one chef before scaling.
  • Role split (one pattern among many): cooks on stoves, choppers on boards, runner for plating, delivery, and bussing.
  • Sort work by order.timeLeft / urgency.
  • Automate the plate lifecycle early—most naive agents die when the rack empties.
  • Prefer onTick over raw setInterval so logic stays aligned with the simulation step.
  • Guard every command with game state checks and chef availability to avoid error spam.

8. Example Agents

These are teaching examples—not optimal solutions. They omit multi-recipe routing, full bussing, boost timing, and rush spikes.

Example 1: Single-chef steak bot

Uses plating_0 as temporary storage for cooked meat, then fetches a plate and merges at the same plating area before delivering to the stand id from the Steak order.

KitchenAPI.onTick(() => {
  const s = KitchenAPI.getState();
  if (!s.running || s.paused) return;
  const c = s.chefs[0];
  if (c.busy || c.hasPath) return;
  const steak = s.orders.find(o => o.dish === 'Steak');
  if (!steak) return;

  const h = c.holding;
  const platItems = s.stations.platingAreas[0].items || [];
  const meatReady = platItems.some(
    i => i.ingredient === 'meat' && i.state === 'cooked' && i.type !== 'plate'
  );

  if (!h) {
    KitchenAPI.command(0, meatReady ? 'dishrack_0' : 'bin_3');
    return;
  }
  if (h.ingredient === 'meat' && h.state === 'raw') {
    KitchenAPI.command(0, 'stove_0');
    return;
  }
  if (h.ingredient === 'meat' && h.state === 'cooked') {
    KitchenAPI.command(0, 'plating_0');
    return;
  }
  if (h.type === 'plate') {
    const done = h.items && h.items.some(
      i => i.ingredient === 'meat' && i.state === 'cooked'
    );
    KitchenAPI.command(0, done ? steak.standId : 'plating_0');
  }
});

Example 2: Simple multi-chef coordinator

Chef 0 cooks meat; Chef 1 chops lettuce for Salad; Chef 2 plates and delivers the most urgent order it can satisfy (very rough heuristic).

function go(id, station) {
  const s = KitchenAPI.getState();
  const c = s.chefs[id];
  if (!s.running || s.paused || c.busy || c.hasPath) return;
  KitchenAPI.command(id, station);
}

KitchenAPI.onTick(() => {
  const s = KitchenAPI.getState();
  if (!s.running || s.paused) return;

  // Chef 0: keep meat flowing to stove_0 when holding raw meat
  const c0 = s.chefs[0];
  if (c0 && !c0.busy && !c0.hasPath) {
    const h = c0.holding;
    if (!h) go(0, 'bin_3');
    else if (h.ingredient === 'meat' && h.state === 'raw') go(0, 'stove_0');
    else if (h.ingredient === 'meat' && h.state === 'cooked') go(0, 'plating_0');
  }

  // Chef 1: chop lettuce for salads when we see a Salad order
  const c1 = s.chefs[1];
  if (c1 && !c1.busy && !c1.hasPath && s.orders.some(o => o.dish === 'Salad')) {
    const h = c1.holding;
    if (!h) go(1, 'bin_1');
    else if (h.ingredient === 'lettuce' && h.state === 'raw') go(1, 'cutting_0');
    else if (h.ingredient === 'lettuce' && h.state === 'chopped') go(1, 'plating_1');
  }

  // Chef 2: plate + deliver — pick urgent order, grab plate, merge at plating_1, walk to stand
  const c2 = s.chefs[2];
  if (!c2 || c2.busy || c2.hasPath) return;
  const urgent = [...s.orders].sort((a, b) => a.timeLeft - b.timeLeft)[0];
  if (!urgent) return;
  const h2 = c2.holding;
  if (!h2) {
    go(2, 'dishrack_0');
    return;
  }
  if (h2.type === 'plate' && (!h2.items || h2.items.length === 0)) {
    go(2, 'plating_1');
    return;
  }
  if (h2.type === 'plate' && h2.items && h2.items.length > 0) {
    go(2, urgent.standId);
  }
});
